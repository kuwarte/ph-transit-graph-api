================================================================================
  1. PROJECT OVERVIEW
================================================================================

  ph-transit-graph is a REST API that digitizes commuter route and fare
  information in the Philippines. Routes, stops, and fares are currently
  scattered, outdated, or unavailable digitally — especially for jeepneys
  and provincial buses.

  This API solves that by:
  - Letting transport operators publish and manage their own routes and fares
  - Letting commuters search routes, find stops, and check fare prices
  - Letting the community flag outdated or incorrect information
  - Letting admins review and approve community corrections

  Think of it like OpenStreetMap but specifically for PH transit fare data,
  backed by a proper REST API with authentication and roles.

================================================================================
  2. GOALS & IMPACT
================================================================================

  REAL-WORLD PROBLEM:
  - No single digital source for jeepney/bus routes and current fares
  - LTFRB fare increases are not reflected anywhere commuters can easily check
  - Commuters, especially students and minimum-wage workers, overpay or get
    shortchanged because they don't know the correct fare

  WHO BENEFITS:
  - Daily commuters who want to know the correct fare before boarding
  - Students navigating unfamiliar routes
  - Transport operators who want a digital presence
  - Developers who can consume this API to build commuter apps

  WHY IT'S PORTFOLIO-WORTHY:
  - Real domain complexity (not a todo app)
  - Multi-role authentication
  - Geospatial data (stop coordinates)
  - Community-driven data governance
  - Covers the full backend stack employers look for

================================================================================
  3. SYSTEM ROLES
================================================================================

  GUEST (no auth)
  - Can search routes, stops, and fares
  - Can view operator profiles
  - Cannot modify anything

  COMMUTER (registered user)
  - Everything a guest can do
  - Can save favorite routes
  - Can flag incorrect fares or stops
  - Can submit new route suggestions

  OPERATOR (transport operator account)
  - Can create and manage their own routes
  - Can add/edit/delete stops on their routes
  - Can set and update fare matrices
  - Cannot touch other operators' data

  ADMIN (platform administrator)
  - Can approve or reject community-flagged corrections
  - Can verify and activate operator accounts
  - Can manage all data on the platform
  - Can view platform analytics

  5. DATABASE ENTITIES & RELATIONSHIPS
================================================================================

  USER
  ─────────────────────────────────────────
  id              UUID (primary key)
  email           VARCHAR UNIQUE NOT NULL
  password        VARCHAR NOT NULL (bcrypt hashed)
  full_name       VARCHAR NOT NULL
  role            ENUM (COMMUTER, OPERATOR, ADMIN)
  is_active       BOOLEAN DEFAULT true
  created_at      TIMESTAMP
  updated_at      TIMESTAMP

  OPERATOR
  ─────────────────────────────────────────
  id              UUID (primary key)
  user_id         UUID (FK → User, one-to-one)
  operator_name   VARCHAR NOT NULL
  franchise_no    VARCHAR UNIQUE         ← LTFRB franchise number
  contact_number  VARCHAR
  is_verified     BOOLEAN DEFAULT false  ← Admin must verify
  created_at      TIMESTAMP

  PLACE
  ─────────────────────────────────────────
  id              UUID (primary key)
  name            VARCHAR NOT NULL          ← e.g. "Pasong Kawayan 2"
  municipality    VARCHAR NOT NULL          ← e.g. "General Trias"
  province        VARCHAR NOT NULL          ← e.g. "Cavite"
  latitude        DECIMAL(9,6)              ← center point of the place
  longitude       DECIMAL(9,6)
  created_at      TIMESTAMP

  ROUTE
  ─────────────────────────────────────────
  id              UUID (primary key)
  operator_id     UUID (FK → Operator)
  origin_id       UUID (FK → Place)         
  destination_id  UUID (FK → Place)         
  origin          VARCHAR NOT NULL
  destination     VARCHAR NOT NULL
  vehicle_type    ENUM (JEEPNEY, BUS, UV_EXPRESS, TRICYCLE)
  status          ENUM (ACTIVE, INACTIVE, UNDER_REVIEW)
  created_at      TIMESTAMP
  updated_at      TIMESTAMP

  STOP
  ─────────────────────────────────────────
  id              UUID (primary key)
  route_id        UUID (FK → Route)
  stop_name       VARCHAR NOT NULL
  sequence_order  INTEGER NOT NULL        ← order of stop along the route
  latitude        DECIMAL(9,6)
  longitude       DECIMAL(9,6)
  landmark        VARCHAR                 ← e.g. "near Puregold"
  created_at      TIMESTAMP

  FARE
  ─────────────────────────────────────────
  id              UUID (primary key)
  route_id        UUID (FK → Route)
  from_stop_id    UUID (FK → Stop)
  to_stop_id      UUID (FK → Stop)
  base_fare       DECIMAL(8,2) NOT NULL
  discounted_fare DECIMAL(8,2)            ← PWD/Senior/Student fare
  effective_date  DATE NOT NULL
  created_at      TIMESTAMP

  SAVED_ROUTE (commuter bookmarks)
  ─────────────────────────────────────────
  id              UUID (primary key)
  user_id         UUID (FK → User)
  route_id        UUID (FK → Route)
  saved_at        TIMESTAMP

  REPORT (community flagging)
  ─────────────────────────────────────────
  id              UUID (primary key)
  reporter_id     UUID (FK → User)
  route_id        UUID (FK → Route, nullable)
  stop_id         UUID (FK → Stop, nullable)
  fare_id         UUID (FK → Fare, nullable)
  report_type     VARCHAR                 ← WRONG_FARE, WRONG_STOP, etc.
  description     TEXT NOT NULL
  status          ENUM (PENDING, APPROVED, REJECTED)
  reviewed_by     UUID (FK → User, nullable)
  reviewed_at     TIMESTAMP
  created_at      TIMESTAMP

  RELATIONSHIPS DIAGRAM:
  ─────────────────────────────────────────

  User ──(1:1)──> Operator
  Operator ──(1:N)──> Route
  Route ──(1:N)──> Stop
  Route ──(1:N)──> Fare
  Stop ──(N:1)──< Fare (from_stop and to_stop)
  User ──(N:M)──> Route (via SavedRoute)
  User ──(1:N)──> Report
  Place ──(1:N)──> Route (as origin)
  Place ──(1:N)──> Route (as destination)

  6. FULL API ENDPOINTS LIST
================================================================================

  BASE URL: http://localhost:8080/api/v1

  Legend:
  [PUBLIC]   = No auth needed
  [COMMUTER] = Requires COMMUTER or higher role JWT
  [OPERATOR] = Requires OPERATOR role JWT
  [ADMIN]    = Requires ADMIN role JWT

  ── AUTH ─────────────────────────────────────────────────────────────────────

  POST   /auth/register              [PUBLIC]    Register new commuter account
  POST   /auth/login                 [PUBLIC]    Login, returns JWT token
  POST   /auth/register/operator     [PUBLIC]    Register as operator (pending verify)
  GET    /auth/me                    [COMMUTER]  Get current logged-in user info
  POST   /auth/refresh               [COMMUTER]  Refresh JWT Token

  ── PLACES ───────────────────────────────────────────────────────────────────

  GET    /places                        [PUBLIC]   Get all places
  GET    /places/{id}                   [PUBLIC]   Get a single place
  GET    /places/search                 [PUBLIC]   Search places
                                                 Query params: ?municipality=&province=
  GET    /places/{id}/routes            [PUBLIC]   Get all routes from/to this place
  POST   /places                        [ADMIN]    Add a new place
  PUT    /places/{id}                   [ADMIN]    Update a place
  DELETE /places/{id}                   [ADMIN]    Delete a place

  ── ROUTES ───────────────────────────────────────────────────────────────────

  GET    /routes                     [PUBLIC]    Get all active routes (paginated)
  GET    /routes/{id}                [PUBLIC]    Get a single route by ID
  GET    /routes/search              [PUBLIC]    Search routes by origin/destination
                                                 Query params: ?origin=&destination=
  GET    /routes/code/{routeCode}    [PUBLIC]    Get route by route code
  GET    /routes/vehicle/{type}      [PUBLIC]    Filter routes by vehicle type
  POST   /routes                     [OPERATOR]  Create a new route
  PUT    /routes/{id}                [OPERATOR]  Update own route
  DELETE /routes/{id}                [OPERATOR]  Deactivate own route
  GET    /routes/my-routes           [OPERATOR]  Get all routes owned by operator
  PATCH  /routes/{id}/status         [ADMIN]     Change route status

  ── STOPS ────────────────────────────────────────────────────────────────────

  GET    /routes/{routeId}/stops           [PUBLIC]    Get all stops for a route
  GET    /routes/{routeId}/stops/{stopId}  [PUBLIC]    Get a specific stop
  POST   /routes/{routeId}/stops           [OPERATOR]  Add a stop to route
  PUT    /routes/{routeId}/stops/{stopId}  [OPERATOR]  Update a stop
  DELETE /routes/{routeId}/stops/{stopId}  [OPERATOR]  Remove a stop
  PATCH  /routes/{routeId}/stops/reorder   [OPERATOR]  Reorder stops sequence

  ── FARES ────────────────────────────────────────────────────────────────────

  GET    /routes/{routeId}/fares              [PUBLIC]    Get full fare matrix for route
  GET    /fares/calculate                     [PUBLIC]    Calculate fare between two stops
                                                          Query params: ?from=&to=
  POST   /routes/{routeId}/fares              [OPERATOR]  Add fare entry
  PUT    /routes/{routeId}/fares/{fareId}     [OPERATOR]  Update fare entry
  DELETE /routes/{routeId}/fares/{fareId}     [OPERATOR]  Delete fare entry

  ── OPERATORS ────────────────────────────────────────────────────────────────

  GET    /operators                  [PUBLIC]    Get all verified operators (paginated)
  GET    /operators/{id}             [PUBLIC]    Get operator profile
  GET    /operators/{id}/routes      [PUBLIC]    Get all routes by an operator
  PUT    /operators/profile          [OPERATOR]  Update own operator profile
  GET    /operators/pending          [ADMIN]     Get unverified operator accounts
  PATCH  /operators/{id}/verify      [ADMIN]     Verify an operator account

  ── COMMUTER FEATURES ────────────────────────────────────────────────────────

  GET    /commuter/saved-routes            [COMMUTER]  Get saved/bookmarked routes
  POST   /commuter/saved-routes/{routeId}  [COMMUTER]  Save a route
  DELETE /commuter/saved-routes/{routeId}  [COMMUTER]  Remove saved route

  ── REPORTS (Community Flagging) ─────────────────────────────────────────────

  POST   /reports                    [COMMUTER]  Submit a flag/report
  GET    /reports/my-reports         [COMMUTER]  View own submitted reports
  GET    /reports                    [ADMIN]     Get all reports (filterable by status)
  GET    /reports/{id}               [ADMIN]     Get a single report detail
  PATCH  /reports/{id}/approve       [ADMIN]     Approve a report (triggers data fix)
  PATCH  /reports/{id}/reject        [ADMIN]     Reject a report with reason

  ── ADMIN ────────────────────────────────────────────────────────────────────

  GET    /admin/users                [ADMIN]     List all users (paginated)
  GET    /admin/users/{id}           [ADMIN]     Get user detail
  PATCH  /admin/users/{id}/deactivate [ADMIN]   Deactivate a user account
  GET    /admin/analytics            [ADMIN]     Platform stats
                                                 (total routes, operators, reports)

================================================================================
  QUICK REFERENCE — HTTP STATUS CODES TO USE
================================================================================

  200 OK              → Successful GET, PUT, PATCH
  201 CREATED         → Successful POST (resource created)
  204 NO CONTENT      → Successful DELETE
  400 BAD REQUEST     → Validation failed, malformed input
  401 UNAUTHORIZED    → No token or invalid token
  403 FORBIDDEN       → Valid token but wrong role
  404 NOT FOUND       → Resource doesn't exist
  409 CONFLICT        → Duplicate resource (email, route code)
  500 INTERNAL ERROR  → Unexpected server error

================================================================================
  QUICK REFERENCE — REST URL CONVENTIONS
================================================================================

  Use nouns, not verbs in URLs:
  ✅ GET    /routes              → list all routes
  ✅ GET    /routes/{id}         → get one route
  ✅ POST   /routes              → create a route
  ✅ PUT    /routes/{id}         → full update a route
  ✅ PATCH  /routes/{id}/status  → partial update
  ✅ DELETE /routes/{id}         → delete a route

  ❌ GET    /getRoutes           → wrong (verb in URL)
  ❌ POST   /createRoute         → wrong (verb in URL)
  ❌ GET    /routes/getById/1    → wrong (redundant)
